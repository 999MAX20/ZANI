- API evidence: `/api/deals/` returns the manager-owned deal, but `/api/pipelines/` and `/api/pipeline-stages/` return empty lists for the manager.
- Impact: manager can have deals but cannot operate the pipeline board properly.
- Fix: shared CRM configuration entities `Pipeline`, `PipelineStage`, and `StageTransition` now remain business-scoped when the user has deal view access; row-level scope still applies to actual deal records.
- Retest: manager now gets 1 pipeline, 1 stage, 1 owned deal and `/dashboard/deals` shows `Role QA Sensitive Deal` without the empty-pipeline warning.
